You are optimizing context token usage for an AI agent system.

## Problem Statement
An autonomous AI agent system loads extensive context at startup (STATE.md, governance contracts, documentation). The current implementation uses 15,000+ tokens per session, causing:
1. High API costs
2. Slower response times
3. Reduced effective context window for actual work

You need to reduce token usage without losing critical information that agents need.

## Available Context
- STATE.md contains 8,500 tokens of implementation status
- 40% of STATE.md is "DONE" items (historical, rarely consulted)
- 30% is "BLOCKED" items (informational, not actionable)
- 30% is "IN PROGRESS" or "TODO" (critical for agents)
- Agents consult STATE.md 2-3 times per session
- False negatives (missing critical info) are more expensive than false positives
